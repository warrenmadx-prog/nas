What is Samba and Why Use It?

Samba is free, open-source software that allows different computers to share files over a network. Think of it like your own personal Dropbox or Google Drive, but you control everything on your own home network.

Perfect for: Accessing your external hard drive full of movies, music, or video projects from any computer in your home - no more unplugging and moving the USB drive around!

The Main Use Case: External USB Drive Sharing

The Problem This Solves

You have a 4TB USB hard drive connected to your Linux Mint desktop with:

  • Your entire media collection (movies, music, shows)
  • Your game library (Steam games, ROMs, installers)
  • Large project files (video editing, 3D models, photos)
  • Important documents and backups

You want to:

Stream media or play games from your laptop

Access large files without moving the USB drive

Edit projects using files stored on the drive

Keep everything centralized in one place

This is what Samba does perfectly!

Part 1: Setting Up the Host (The Computer with the USB Drive)

Step 1: Find Your USB Drive

First, we need to know where Linux Mint mounted (connected) your USB drive.

Find your username first:

whoami

This shows your username. Write it down! We'll use it throughout this guide.

Example output: john (this would be your actual username)

Wherever you see username in this guide, replace it with this!

Plug in your USB drive, then open Terminal and type:

lsblk

What you'll see: A list of all drives and partitions. Look for something like:

NAME   SIZE  TYPE MOUNTPOINT
sda    4TB   disk
‚îî‚îÄsda1 4TB   part /media/kim/MyPassport

The mount point is what appears under MOUNTPOINT. In Linux Mint, it's usually:

  • /media/[your-username]/[drive-name]
  • Example: /media/kim/MyPassport
  • Example: /media/kim/USB-DRIVE

Easy way to see it: Open the File Manager (Nemo), and your USB drive appears in the left sidebar. Right-click it and select "Properties" to see the location.

Write this down! This is the path we'll share. For this guide, our example USB drive is at: /media/kim/external

Step 2: Check the Drive is Accessible

Let's make sure you can actually see the files:

ls -la /media/kim/external

Replace /media/kim/external with your actual USB drive path!

What you'll see: A list of folders and files on your USB drive, like:

drwxr-xr-x  5 kim kim  4096 Dec 18 10:30 Media
drwxr-xr-x  3 kim kim  4096 Dec 15 09:15 Games
drwxr-xr-x  2 kim kim  4096 Dec 10 14:22 Projects
drwxr-xr-x  4 kim kim  8192 Dec 05 11:30 Documents

If you see "Permission denied": We'll fix permissions in the next step.

Step 3: Set Proper Permissions

Your USB drive needs the right permissions so Samba can share it.

sudo chmod 755 /media/username/external
sudo chown username:username /media/username/external

CRITICAL: Replace these with YOUR actual information!

Example if your username is john and drive is at /media/john/MyPassport:

sudo chmod 755 /media/john/MyPassport
sudo chown john:john /media/john/MyPassport

How to find your username if you forgot: Type whoami in the terminal

Breaking it down:

  • chmod 755: Sets permissions
    • 7 = You can read, write, and access folders
    • 5 = Others can read and access, but not modify
  • chown username:username: Makes your user account the owner
    • First username = the user
    • Second username = the group (usually the same as your username)

For external drives formatted as NTFS or FAT32: You might need different options. If you have write permission issues later, we'll troubleshoot that.

Method 1: Using File Manager (Nemo/Nautilus) - Easiest!

This is the simplest way to access your shared drive.

  1. Open File Manager (folder icon in taskbar)
  2. Look in the left sidebar and click "Network"
  3. If you see your host computer, double-click it
  4. If not, press Ctrl + L and type: smb://192.168.1.105/SharedDrive
  5. Enter your Samba username and password when prompted

Success! The USB drive now appears in your sidebar under "Network".

Windows Client

Method 1: Quick Connect

  1. Open File Explorer (Windows Key + E)
  2. Click in the address bar at the top and type your host's static IP
  3. Type: \\192.168.1.11
  4. Press Enter and enter your credentials

Method 2: Map as Network Drive (Recommended)

This makes the USB drive appear as a permanent drive letter (like Z:).

  1. Open File Explorer
  2. Click "This PC" in the left sidebar
  3. Click "Map network drive" in the toolbar
  4. Choose a drive letter (like Z:)
  5. Type: \\192.168.1.105\SharedDrive in the Folder field
  6. Check "Reconnect at sign-in" and "Connect using different credentials"
  7. Click Finish and enter your Samba credentials

  2. Press Cmd + K (or Go ‚Üí Connect to Server)

Use Case C: Backing Up Files (All Platforms)

From Windows

Simple drag-and-drop:

  1. Open File Explorer
  2. Navigate to your mapped network drive (like Z:)
  3. Drag folders from your computer to the network drive

From Linux (Automatic backup with rsync)

Create a simple backup script:

#!/bin/bash
rsync -av --delete /home/user2/Documents/ /mnt/SharedDrive/Backups/Documents/
rsync -av --delete /home/user2/Pictures/ /mnt/SharedDrive/Backups/Pictures/
echo "Backup completed on $(date)" >> /home/user2/backup.log

Note: Replace user2 with YOUR actual Linux username on the client computer!

Troubleshooting Common Issues

Problem: "Failed to mount" or "Connection refused"

Test network connectivity:

ping 192.168.1.105

Replace 192.168.1.105 with YOUR host's actual IP address!

If you get replies: Network is fine, keep troubleshooting

If no reply:

  • Check both computers are on same WiFi
  • Verify the host computer is powered on
  • Check the IP address didn't change (run hostname -I on host again)
Problem: "Permission denied" or "Access denied"

On the host computer, check Samba users:

sudo pdbedit -L

If your user isn't listed:

sudo smbpasswd -a username
sudo smbpasswd -e username

Try resetting the password:

sudo smbpasswd username
Problem: Video stutters or buffers / Games lag constantly

Quick fixes:

  • Close other applications using network (downloads, updates)
  • Move closer to WiFi router
  • Switch to 5GHz WiFi (if available)
  • Try VLC for media or different game if gaming
  • Test with lower resolution media or less demanding game

Better solution:

  • Use Ethernet cable on both computers (essential for gaming)
  • Upgrade to better WiFi router
  • Use WiFi 6 if both computers support it
  • For games: Copy game to local drive for best performance
